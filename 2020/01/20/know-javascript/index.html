Know JavaScript

Posted on 2020-01-20 Edited on 2020-04-29

This is a JavaScript guide for developers who already know other programming languages.

What it is

  • Untyped, prototype-based, single-threaded scripting language
  • Native Unicode (UTF-16)
  • Case-sensitive
  • First-class functions (support functional programming styles)
  • A minimal API for working with text, arrays, dates, and regular expressions, but no Input/Output, networking, storage, and graphics, these are supported by web browsers.

Identifiers

Begin with letter, _, $, then followed by letters, digits, _, $

Comments

  • //, for single line
  • /* ... */, for single line or multiple lines

Statements

The semicolon (;) is used to separate statements, it is optional, but no surprising cases when using it

Data types

  • Primitive types

    • number
    • string
    • boolean
    • null
    • undefined
    • Symbol
    • BigInt

    They are immutable and are compared by value

  • Object types

    • array and function are special objects;

    They are mutable and are compared by reference

number

Array indexing and the bitwise operators are 32-bit integers, All other numbers are represented as 64-bit binary floating-point values(IEEE-754), it can exactly represent 1/2, 1/8, ..., 1/2^n, but can't exactly represent 1/10, 1/100, they are decimal fractions.

Arithmetic operators:

  • +, -, *, /, %, **, ++, --
  • Math object: Math.log(100)

Doesn't raise overflow, underflow, division by zero errors, but the arithmetic results are special values:

  • overflow: Infinity / -Infinity
  • underflow: 0 / -0
  • division by zero: Infinity / -Infinity, but 0 / 0 is NaN(not a number)

Infinity, NaN are global variables. Infinity represents infinity. NaN represents not-a-number.

To check x is NaN: x != x, it is true if only if x is NaN, don't use x == NaN.

string

Enclose a string with single or double quotes (' or "). When combining JavaScript and HTML, one style of quotes for JavaScript, the other for HTML.

A string likes a read-only array, could use index to access.

Template strings

A template string is enclosed by the backtick (` `). The dollar sign and curly braces (${expression}) indicate a placeholder.

Tagged templates

More details here:
https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Template_literals

boolean

Any values can be converted to a boolean value: undefined, null, 0, -0, NaN, "" are converted to false, others are true.

null

null is an object, indicates "no value".

undefined

Get an undefined value when access an uninitialized variable, a non-existed object property or array element, a function argument that you don't supply a value, the return value of a function that have not return value.

undefined is a error-like absence of value, if you want a variable has not any value, assign null to it.

Wrapper objects

Numbers, strings, booleans are not objects, but they have properties and methods, because of JavaScript converts them to an object when refer the value of a property or invoke a method, then discard the object.

String(), Number(), Boolean() return wrapper objects.

Variables

Variable declaration

  • var

Declares a local or global variable, optionally initializing it to a value.

  • let

Declares a block-scoped variable, optionally initializing it to a value.

  • const

Declares a block-scoped constant, the value of a constant can't be changed through reassignment.

nameN can be any legal identifier;

valueN can be any legal expression, including a function expression;

How to choose:

let/const vs var:

  • Can't be redeclared
  • Declared variables outside functions or blocks aren't a property of the global object
  • Can't be used before declaring it

const vs let:

  • Must be initialized
  • Can't be reassigned

If the value of a variable won't be changed, use const, otherwise use let whenever possible. The post use many variables declared by var before I think and know the difference.

Prototype and inheritance

All constructor functions have a prototype property, the property is a template object for storing properties and methods that is used to build the __proto__ property of a new object when creating the object with the new operator, making the new object is an instance of the current object. Any object has the __proto__ property, or the object returned by Object.getPrototypeOf(), it is a part of the prototype chain, used to look up properties and methods.

A fairly common pattern for more object definitions is to define the properties inside the constructor, and the methods on the prototype. if defining a method in the constructor, it is a closure, the methods would get reassigned for every object creation, might have performance issues, more details here.

Bitwise operators

&, |, ^, ~, <<, >>, >>>

The bitwise operators expect 32-bit integer operands, and drop any fractional part and any bits beyond the 32nd.

NaN, Infinity, and -Infinity all convert to 0.

Comparison operators

==, ===, !=, !==

===, !== is the strict version of ==, !=, they check the type of operands first, then compare values:

  • different types, not equal
  • one or both values is NaN, is not equal
  • 0 === -0 is true
  • strings are Unicode (UTF-16) strings, two equal strings must have a same length and same UTF-16 bit sequences
  • objects are compared by reference, two equal objects must refer to a same object

==, != perform some type conversions, then compare, the rules:

  • if two values have the same type, do same things like ===, !==, if not, check the following rules;
  • null == undefined is true
  • one is number, one is string, convert the string to a number, then compare
  • one of values is boolean, convert the boolean to a number, true to 1, false to 0, then compare
  • one of values is object, perform object-to-primitive conversions(favor number, then string), then compare
  • any other combinations are not equal

<, >, <=, >=

These operators favor numbers and only perform string comparison if both operands are strings, don't like +(favors strings).

The rules:

  • one of values is object, perform object-to-primitive conversions first;
  • both are strings, compare as strings;
  • one of values is not string, converts all of them to numbers, then compare
  • Infinity is larger any number, -Infinity is the opposite
  • either of operand is NaN, the result is false

in operator

The left-side operand is a string, the right-side operand is an object. It is true if the left-side value is a property name of the object.

instanceof operator

The left-side operand is an object, the right-side operand is the constructor function to initialize the object.

Logical operators

&&, ||, ~

~ operator converts its operand to a boolean value first, but && and || don't

eval()

It is a global function which interpret the string as JS source code, return the value of the last expression or statement or undefined if no return value like in a function.

If calling eval() within a function, it uses the environment of the caller, can do anything like the caller within the function.

If calling eval() outside functions, it use the global environment.

The source code in eval() is unanalyzable, could assign eval to a variable, can't safely optimize functions that call eval().

If assigning eval to a variable within a function, it is a indirect call to eval, eval uses the global environment, not the local environment, can't read, write, define local variables or functions.

This operator is frequently used as a shortcut for the if statement.

Explicit conversions

Primitive-to-Object conversions

The Primitive-to-object conversions are straightforward, using wrapper objects: String(), Number(), Boolean().

No conversion and raise TypeError when using null and undefined where an object is excepted.

Object-to-Primitive conversions

All objects convert to the boolean true.

All objects(don't include web browsers defined objects) inherit toString() and valueOf(), could override them in custom objects.

The toString() returns a string representing the object

The valueOf() returns the primitive value of the object

The rules:

  • To convert an object to a string

    • check toString() returned value, overrided toString() can return any value, if the value is primitive, convert it to a string(if necessary), done;
    • check valueOf(), if the value is primitive, convert it to a string(if necessary), done;
    • throw TypeError

  • To convert an object to a number

    • similar to string, but check valueOf() first, then toString();

The object-to-primitive conversion is an object-to-number conversion first( valueOf() ), then an object-to-string conversion( toString() ).

+, -, *, /, %, ==, !=, <, > operators use the object-to-primitive conversion, use the returned primitive value directly, no further conversion to a number or string.

If either of operands of + is object, perform the object-to-primitive conversion, then if one of the operands is string, perform string concatenation, otherwise, is arithmetic. Look at the difference of line 1, line 11.

Date objects are special, its +, ==, != use an object-to-string conversion( toString() ) first, not valueOf() first;

for...in

Iterates over all enumerable properties of an object, including inherited enumerable properties.

Think the object is a list of key-value pairs, for...in return the keys, not the values.

variable must be an lvalue, may be an existing variable, a var/let/const declared new variable, or any expression that evaluates to an lvalue.

object can be any expression that evaluates to an object, do type conversions, convert the primitive value to its wrapper object.

Only enumerate enumerable properties of an object, not all properties. All properties and methods defined by our code are enumerable.

Iterate over the properties in an arbitrary order, don't depend on the order you see, especially to use with an array and may be return non-integer property names(not recommended for array)

Strict mode

Use 'use strict' to invoke the strict mode.

"use strict" or 'use strict' is a directive, not a statement, appears only at the top of a JS script or a function body before any real statements.

Don't do this:

  • concatenating strict code, non-strict code, the result is strict;
  • concatenating non-strict code, strict code, the result is non-strict;

JavaScript modules are strict by default.

Code passed to eval() is in strict mode if calling eval() (must call directly) from strict code. Can also use 'use strict' directly in eval().

Why to use

  • throwing errors instead of some silent errors, fix problems easier;

  • making things simpler, let JS engine optimize the code better;

  • write more secure JavaScript code;

  • Making eval and arguments simpler

As tree of nodes

Use properties of the Node object:

  • parentNode
  • childNodes

  • previousSibling

As tree of elements

Use properties of the Element object:

Text and comment nodes are not elments, they are ignored.

The <h2> element is in the traversal path, its text "Properties" won't be.

Element attributes

The Element object has an attributes property which is a live collection including all attributes of the element. Also have a getAttributeNames() methods which returns all attribute names.

Standard HTML attributes

These attributes are properties of an element, straightforward.

HTML attributes are not case sensitive, but JavaScript isn't, the property names are lowercase. A few HTML properties are longer than one word, such as http-equiv, convert them to the style: propNameLikeThis.

If HTML attribute names are JavaScript reserved words, prefixing them with html, but class is an exception, it is className, not htmlClass.

Non-standard HTML attributes

Use methods:

  • getAttribute()
  • hasAttribute()
  • setAttribute()
  • removeAttribute()

getAttribute() returns a string, or null, or "".

Use hasAttribute() to check for an attribute's existence, not the return value of getAttribute().

setAttribute() always converts its value argument to a string. Doesn't care about boolean attributes' actual value, they're present, they're true, we can just set their value to "".

removeAttribute() can be used to remove a standard attribute, but just empties its value in Javascript, the HTML attribute in the web page will disappear.

data-* attributes

HTML5 defines a standard way to attach additional data, any attribute whose name is lowercase and begins with the prefix data- is a valid custom data attribute. e.g.

Valid custom data attribute names can only include lowercase letters, numbers, dash(-), underscore(_), dot(.), colon(:).

The property name in JavaScript is abc:123Efg_H32.hi, remove data- and dashs, the letters that follow a dash are transformed into its uppercase.

The dataset property is a DOMStringMap which stores all custom data attributes of the element and is live, the property itself is read-only, but can read, write, delete its members.

Each property in dataset is always a string.

Notes:

  • Do not store content that should be visible and accessible in data attributes.
  • Poor read performance.

More datails here:

Element content

As HTML

  • innerHTML property, gets or replaces the HTML code contained within the element
  • outerHTML property, like innerHTML, but apply to the element itself and its descendants
  • insertAdjacentHTML() method, insert HTML code

If an element has no parent element, setting its outerHTML property does nothing.

The markup and formatting of the returned HTML by innerHTML or outerHTML is likely not to match the original page markup.

HTML5 doesn't execute <script> tag in innerHTML and outerHTML, but there are still other ways to execute JavaScript, don't use them to insert untrusted HTML code or plain text (use textContent property).

More details here:
https://developer.mozilla.org/en-US/docs/Web/API/Element/innerHTML
https://developer.mozilla.org/en-US/docs/Web/API/Element/outerHTML
https://developer.mozilla.org/en-US/docs/Web/API/Element/insertAdjacentHTML

As plain text

  • textContent property, gets the text content of an element and its descendants, or replaces them with a single text node
  • innerText property, like textContent, but the text is rendered

textContent returns concatenated text which is like innerHTML returned HTML code, but stripping all HTML tags, so ignores <br>, a newline is gone, but preserves tabs, spaces that are within these tags. It gets the content of all elements, including <script> and <style> elements.

innerText returns concatenated text which is like we highlight an area of a web page with the cursor and then copy it, only shows human-readable elements, no <script>, <style> and hidden (e.g. display:none) elements, these elements are not rendered.

The returned values by textContent and innerText are same if the element is not rendered, such as:

More details here:
https://developer.mozilla.org/en-US/docs/Web/API/Node/textContent

Text in <script>

Could embed arbitrary text into <script>, just set an invalid MIME type, make the script non-executable.

Element coordinates

Document Coordinates: a point (x, y) of an element is relative to the top-left corner of the document.

Viewport Coordinates: a point (x, y) of an element is relative to the top-left corner of the viewport, the viewport is the portion of the browser window that actually displays document content, excluding browser menus, toolbars.

width, height of the document

width, height of the viewport

Properties in the Window object:

  • innerWidth
  • innerHeight

The two properties' value include scrollbars and borders.

  • scrollX (alias: pageXOffset, for compatibility)
  • scrollY (alias: pageYOffset, for compatibility)

Return the number of pixels that the document is currently scrolled horizontally or vertically.

Methods in the Window object:

  • scroll()
  • scrollBy()
  • scrollTo()

Properties in the Element object:

  • clientWidth
  • clientHeight

Return inner width, height of an element, including padding but excluding borders, margins, and scrollbars (if present). scrollbars' position are between padding and borders;

When they are used on the root element, they are width, height of the viewport, excluding scrollbars;

They return 0 for elements with no CSS or inline layout boxes.

  • scrollWidth
  • scrollHeight

Minimum required width, height of an element to show without scroolbars, measured like clientWidth, clientHeight, but including width, height of pseudo-elements;

Don't need a horizontal scrollbar if clientWidth == scrollWidth, the vertical scrollbar is also.

  • scrollLeft
  • scrollTop

Get or set the number of pixels that an element's content is scrolled horizontally or vertically.

Methods in the Element object:

  • scroll()

  • scrollBy()

  • scrollIntoView(). Also could mark an anchor, then change location.hash to it

  • scrollTo()

  • getBoundingClientRect()

    Returns the size of an element and its position relative to the viewport. The position includes padding and borders, not margins. The returned value is the union of the rectangles returned by getClientRects().

    Convert the viewport coordinates to the document coordinates:

  • getClientRects()

Like getBoundingClientRect(), but returns a collection. Inline elements may consist of multiple rectangles, use it to query these individual rectangles.

  • elementFromPoint()
  • elementsFromPoint()

Return the element or elements at the specified coordinates (relative to the viewport).

Properties in the HTMLElement object:

  • offsetWidth
  • offsetHeight

Return width, height of an element, include padding, borders, scrollbars, not margins. They are opposite to clientWidth and clientHeight;

scrollWidth > offsetWidth if an content overflow is happening.

  • Imported stylesheet

The styles which reside closest to the HTML tag will take the precedence, the above sequence is also the precedence order, JavaScript generated styles are same as the inline stylesheet.

Comments

Comments in CSS begin with /* and end with */

Margin and padding

  • margin specifies space outside the border, provides visual space between an element and its neighbors in the normal document flow;
  • padding specifies space inside the border, between the border and the element content;
  • no border, padding is not necessary. margin is irrelevant if the element is dynamically positioned;

How to scripting CSS

With the style property

The property is used to get as well as set the inline style of an element. The returned object is read-only, but can set value to its properties, all values must be string. Getting or setting the inline style as a single string to use elt.setAttribute() or elt.style.cssText.

Will get nothing if the element hasn't inline styles, only has value if setting it by JavaScript code before or the element has inline styles. in general, only useful for setting styles.

Setting a property of the style object to null or "" will reset it, prefer "".
